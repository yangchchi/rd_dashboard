# HAI 智研平台 — 产品简介与功能清单

> 本文档描述本仓库所实现的 **HAI 智研平台**（研发代号：`rd_dashboard`）的产品定位、用户价值与功能范围，便于对内对外说明与评审对齐。

---

## 一、产品简介

**HAI 智研平台**（界面品牌名，副标题 *AI-Driven SDLC*）是一套 **AI 驱动型研发管理系统** 原型与可运行实现，面向从「原始想法」到「可验收交付」的全链路协作。系统以 **需求** 为主线，串联 **PRD、技术规格、AI 流水线、测试与验收**，用统一的数据视图与权限模型，降低干系人、产品经理与技术经理之间的信息断层。

### 1.1 产品类型与场景

| 维度 | 说明 |
|------|------|
| 产品形态 | 多页面 Web 应用（Sidebar 导航），前后端一体化 |
| 典型场景 | 企业内研发协同、AI 辅助文档与规格产出、流水线状态与质量可视化 |
| 界面语言 | 中文为主 |
| 工程实现 | 前端 Next.js（App Router）+ 后端 NestJS + PostgreSQL（Drizzle ORM） |

### 1.2 核心价值

1. **链路透明**：需求在多个研发阶段的状态可在一处查看与推进，减少「需求在哪一环节、谁在负责」的反复沟通。
2. **文档与规格结构化**：PRD 与技术规格以可编辑、可关联需求的形态管理，为下游 AI 与工程消费提供一致输入。
3. **AI 能力网关化**：AI 技能通过服务端能力插件配置与调用，避免密钥暴露在前端，便于审计与替换模型。
4. **角色与权限**：菜单与路由级访问控制，同一套界面按角色收敛可见功能，贴合干系人 / PM / TM 的分工。

### 1.3 目标用户

| 角色 | 说明 |
|------|------|
| 干系人（Stakeholder） | 提交与验收需求，关注业务结果与上线节奏 |
| 产品经理（PM） | 维护 PRD、推进评审与阶段流转 |
| 技术经理（TM） | 规格定义、流水线与质量、技术决策 |

---

## 二、研发全生命周期（业务主线）

平台将单条需求在研发过程中的位置抽象为 **多阶段状态**（与看板泳道、列表筛选一致），典型路径包括：需求池 → PRD 编写 → 规格定义 → AI 开发 → 待验收 → 已发布等（具体状态以系统内枚举与看板为准）。  
主线数据通过全局上下文键（如当前需求、PRD、规格列表缓存等）在 **智研看板、需求详情、PRD/规格编辑、流水线、验收** 等页面之间传递与延续编辑会话。

## 三、功能清单

以下按 **用户可见模块** 与 **平台治理能力** 归纳；标「规划/插件」项以 `AGENTS.md` 与 `server/capabilities/*.json` 中的能力配置为准，实际交互以界面为准。

### 3.1 智研看板

- 多列泳道展示需求在各阶段的分布。
- 需求卡片展示标题、优先级、时间、负责人等关键信息，支持进入详情。
- 阶段统计与筛选（如「我提交的 / 我负责的 / 全部」等，以页面实现为准）。
- 支持拖拽等交互以调整需求阶段（受业务规则约束）。

### 3.2 需求中心

- **需求列表**：表格或列表视图，支持搜索、筛选与批量相关能力（以当前实现为准）。
- **需求采集**（`/requirements/new`）：标准化录入需求描述、优先级、期望时间等；可含草图/附件与 AI 辅助分类等能力。
- **变更类型（杯子模型）**：首问区分 **Greenfield**（新做一个产品/大功能）与 **Brownfield**（增强 / 缺陷 / 重构）；存量改动须绑定 **所属产品** 与 **产品基线**。
- **需求详情**（`/requirements/[id]`）：基础信息、关联 PRD/规格/分支等入口、流转历史与时间轴、按状态展示可操作按钮；展示变更类型与基线版本标签。
- **需求编辑**（`/requirements/[id]/edit`）：对已存在需求的修改入口；可调整变更类型、产品与基线绑定。

### 3.3 智能文档（PRD）

- **PRD 列表**：展示 PRD 与关联需求、状态、更新时间等。
- **PRD 编辑**（`/prd/[id]/edit`）：分区块编辑背景、目标、功能列表、非功能需求等；可含流程图/版本与评审状态字段（以页面为准）。
- **Delta PRD（Brownfield）**：对增强/缺陷/重构类需求，AI 生成与编辑侧重「基线引用 / 本次变更 / 不变更声明 / 影响面」四段增量结构，避免重写整份存量 PRD；编辑页展示所引用的产品基线版本与产品 Hub 链接。
- **AI 生成 PRD（插件：prd_generator）**：基于选定需求生成结构化 PRD 草稿，流式输出（stream）；Brownfield 时 prompt 注入 As-Built 摘要与 Delta 约束。

### 3.4 技术基准（规格定义）

- **规格列表**：按 PRD 或产品维度查看 FS/TS 完成情况。
- **规格编辑**（`/specification/[id]/edit`）：功能规格（API、UI 组件、交互）与技术规格（库表、架构、第三方集成等）结构化编辑；支持 Machine-Readable JSON 视图与校验（以页面为准）。
- **冲突检测（插件：conflict_detector）**： unary，用于规格与现有架构/约束的冲突提示（规格页场景）。
- **规格导出**：导出标准化规格产物供 AI 或外部工具消费（以导出按钮/格式为准）。

### 3.5 交付引擎（AI 流水线）

- **流水线看板**：展示各需求在 AI 开发子阶段的状态（生成、自测、构建、部署等语义，以界面文案为准）。
- **任务详情**（`/ai-pipeline/[taskId]`）：单任务日志、代码/工作区面板、质量与指标展示。
- **代码评审助手（插件：code_review_assistant）**： stream，对生成结果做摘要与质量评估类输出。
- **流水线测试用例生成（插件：pipeline_test_case_generator）**： stream，由 FS/TS/代码上下文推导自动化测试用例草案。
- **流水线测试执行报告（插件：pipeline_test_runner）**： stream，基于用例与上下文归纳测试执行报告。
- **人工干预**：暂停、重试、回滚等操作入口（以 TM 权限与后端接口为准）。
- **环境链接**：跳转沙箱/部署环境（以配置与列表为准）。

### 3.6 验收中心

- **待验收列表**：当前用户需验收的需求与访问链接。
- **对比与评分**：需求 vs 实现对比、多维度评分与文字反馈。
- **Brownfield 三列对比**（增强类）：基线能力目录 | 本次 PRD Delta | 交付验证勾选；支持一键生成「相对基线的变更摘要」并复制。
- **RFC/回退**：验收不通过时发起 RFC，需求回退至 PRD 阶段并 **保留 baselineId**；自动跳转 PRD 编辑且提示勿覆盖基线范围。
- **验收通过回写基线（可选）**：Brownfield 验收通过时可勾选将本次 PRD 能力合并冻结为新产品基线版本，供后续需求引用。
- **验收分析（插件：acceptance_feedback_analyzer）**： stream，对验收反馈或基线+Delta 上下文做归纳与改进建议。

### 3.7 赏金猎场

- 与任务验收、积分/代币类激励相关的运营或游戏化模块（具体规则以页面与后端数据模型为准）。

### 3.8 产品主数据与基线（产品 Hub）

- **产品管理**（`/products`）：维护产品线或产品维度主数据（Git、沙箱、生产链接、负责人等）。
- **产品 Hub**（`/products/[id]`）：四 Tab — **概览** / **能力目录** / **产品需求** / **基线历史**；支持冻结 As-Built 基线（version + gitRef + 能力条目）、双版本能力 diff、快捷「新建增强需求」预填产品与基线。

### 3.9 认证与账号

- **登录**（`/login`）：账号登录入口。
- **飞书 OAuth**（`/login/feishu-callback`）：飞书授权回调与会话建立（以环境变量与部署配置为准）。

### 3.10 平台设置与治理

以下入口通常出现在侧栏「设置」分组中，并受 **权限菜单键** 控制可见性：

| 模块 | 路由示例 | 能力概要 |
|------|-----------|----------|
| 组织规格配置 | `/org-spec-config` | 组织级规格/模板等配置 |
| 插件与技能 | `/plugins`、`/skills` | 插件列表与 AI Skill 配置，与 `/api/capability/*` 网关联动 |
| 用户管理 | `/users` | 用户列表与维护 |
| 角色管理 | `/settings/roles` | 角色定义与分配 |
| 权限管理 | `/settings/permissions` | 权限点与策略配置 |
| 通用设置 | `/settings` | 个人或站点级偏好入口（以页面为准） |

### 3.11 全局体验与其它

- **侧栏导航**：品牌区展示「HAI智研平台」与 AI-Driven SDLC 标识；支持折叠与移动端抽屉。
- **主题**：亮色/暗色/跟随系统（`next-themes`）。
- **站内消息**：消息列表与已读标记；未读数量提示（以 `useSiteMessagesList` 等实现为准）。
- **访问控制**：`RequireRouteAccess` 等与后端权限对齐，无权限路由不可达。

### 3.12 需求采集侧 AI（插件）

- **需求分类（插件：requirement_classifier）**： unary，在需求采集页对类型、标签、优先级等做自动建议。

| 插件实例 | 类型基座 | 业务用途 | 输出模式 | 主要页面 |

## 五、技术栈摘要（便于与研发对齐）

- **前端**：React 19、Next.js 15（App Router）、Tailwind CSS、TanStack Query、Radix / Headless UI。  
- **后端**：NestJS 10、能力网关与业务模块（`server/modules`）。  
- **数据**：PostgreSQL + Drizzle ORM，迁移脚本位于 `server/database/migrations/`。  
- **AI 调用**：服务端持有模型 Key，通过环境变量（如 `ARK_API_KEY`）与数据库中的 Skill 配置调用；生产环境应关闭演示模式（`AI_DEMO_MODE`）。
