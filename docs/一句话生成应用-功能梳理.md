# HAI 智研平台 · 一句话生成应用（One-Shot App Generator）功能梳理

> **目标版本**：v1.1
> **功能定位**：平台级 **全局 AI 副驾**，让任意用户在任意页面只输入一句话，即可让大模型即席生成一个"可运行的简单前端应用"，并通过多轮对话持续优化，最终把成果一键沉淀回平台（剪贴板 / 需求中心 / PRD / Agent 工作区）。
> **核心场景**：PM 想快速搭一个交互原型给业务方看；TM 想给一段规格做可视化验证；干系人想给"我想要的样子"举个真实例子，而不是空口描述。

---

## 一、背景与价值

### 1.1 产品背景

当前平台已具备「**需求 → PRD → 规格 → AI 流水线 → 验收**」的完整 SDLC 链路，但所有 AI 能力都是 **页面绑定** 的（PRD 生成只在 PRD 页、规格生成只在规格页）。日常协作中，更高频的场景反而是 **"零碎的、跨页面的、想到就要的"**：

- PM 在写需求时想"我先做个 demo 给业务方看看"
- 干系人验收时想"如果换成另一种交互呢？给我画一下"
- TM 在评审规格时想"这个组件交互的极端值表现是什么样的？"

这些场景下用户需要的不是文档，而是 **"可以点的真实应用"**，且需要 **不打断当前工作流**。

### 1.2 核心价值

| 价值点 | 说明 |
|---|---|
| **零跳转** | 全局悬浮入口，任意页面随时唤起，不打断主任务 |
| **零门槛** | 一句话 → 可点的应用，业务方也能用 |
| **可演进** | 多轮对话持续优化，对话即版本 |
| **可沉淀** | 一键提交到需求 / PRD / Agent 工作区，不再是"用完即弃的玩具" |
| **AI 原生** | 与既有能力网关一致，复用模型、审计、权限、降级体系 |

### 1.3 不在本期范围

- 全栈应用生成（含数据库 / 后端 API） → v1.2 走交付引擎
- 多文件项目脚手架（Vite / Next.js） → v1.2
- 复杂依赖安装 / npm 包动态加载 → 安全风险大，本期仅支持白名单 CDN
- 移动端原生应用生成（React Native 等）

## 二、用户故事

### 2.1 典型用户与场景

| Persona | 场景 | 期望 |
|---|---|---|
| PM 小薇 | 写需求时想给业务方看个原型 | 一句话生成 → 截图发飞书 → 同时挂到需求附件 |
| 干系人小张 | 验收时觉得交互不对劲 | "把这个按钮放右边，加个 loading" → 直接给开发看效果 |
| TM 老李 | 评审规格里某个组件 | "用这个 API mock 数据，给我一个表格 + 筛选" |
| 测试小芳 | 想造一个边界态 | "给我一个 5000 行的虚拟列表" |

### 2.2 用户故事（User Story）

1. **作为 PM**，我希望在任何页面通过悬浮按钮唤起对话，输入一句话获得可运行的前端原型，**以便** 5 分钟内给业务方演示。
2. **作为干系人**，我希望对生成的应用通过自然语言持续修改，**以便** 不懂代码也能精确表达"我想要什么"。
3. **作为 TM**，我希望把对话产物一键提交到需求或 Agent 工作区，**以便** 直接进入下游研发链路。
4. **作为研发**，我希望生成的代码可一键复制为标准片段，**以便** 直接粘到现有项目里。

### 2.3 与现有模块的关系

```text
       ┌─────────────────────── 全局 ───────────────────────┐
       │  悬浮入口（Layout 层）→ 对话面板 → 沙箱预览           │
       └────────────────────────┬──────────────────────────┘
                                │ 沉淀
       ┌────────────┬───────────┼───────────┬────────────┐
       ▼            ▼           ▼           ▼            ▼
   需求中心       智能文档    Agent 工作区   剪贴板      站内消息
  (作为附件)    (插入 PRD)   (生成 PR)    (复制)     (分享链接)
```

## 四、信息架构与交互流

### 4.1 交互流程图

```text
任意页面
   │
   ▼  (点击悬浮按钮 / ⌘+I)
对话面板从右侧滑出
   │
   ├── 首次：展示"推荐 prompt"卡片
   └── 复访：恢复最近会话
   │
   ▼ 输入一句话需求 + (可选) 挂上下文
   │
   ▼ 服务端能力网关流式调用

   │
   ▼  (运行报错 / 想优化)

   │
   ▼ 满意后
```

### 4.2 主要状态

| 状态 | UI 表现 |
|---|---|
| `idle` | 输入框聚焦、Send 可点；预览空状态 |
| `streaming` | 输入框禁用、显示"停止生成"按钮；预览出现进度条 |
| `error` | 红色横幅展示错误，提供"重试 / 改用其他模型 / 让 AI 自动修" |
| `applying` | 收到完整代码后短暂展示"编译应用中"骨架屏 |
| `ready` | 预览可交互，工具栏可用 |

- 悬浮按钮 Tooltip：「一句话生成应用 · ⌘I」

## 五、AI 能力插件设计

### 5.1 新增能力插件：`one_shot_app_generator_1`

放在 `server/capabilities/one_shot_app_generator_1.json`，类型基座 `ai-text-generate`，输出模式 `stream`，与 `prd_generator_1` 同构。

```jsonc
{
  "id": "one_shot_app_generator_1",
  "pluginKey": "@official-plugins/ai-text-generate",
  "pluginVersion": "1.0.0",
  "name": "一句话生成应用",
  "description": "根据一句话需求与多轮对话历史，生成单文件可运行的前端应用（HTML + Tailwind CDN + 受控 JS），支持流式输出。",
  "paramsSchema": {
    "type": "object",
    "properties": {
      "user_intent":           { "type": "string", "description": "用户最新一轮的指令（必填）" },
      "conversation_history":  { "type": "string", "description": "已发生的对话历史（角色 + 文本）" },
      "previous_code":         { "type": "string", "description": "上一版生成的完整代码，用于增量修改" },
      "platform_context":      { "type": "string", "description": "当前页面/需求上下文（可选，自动注入）" },
      "device_target":         { "type": "string", "enum": ["desktop", "tablet", "mobile"], "default": "desktop" },
      "theme":                 { "type": "string", "enum": ["light", "dark"], "default": "light" },
      "framework":             { "type": "string", "enum": ["html-tailwind", "react-tailwind"], "default": "html-tailwind" }
    },
    "required": ["user_intent"]
  }
}
```

### 5.2 Prompt 设计要点

- **强约束输出**：第一个非空字符起即 `<!DOCTYPE html>` 或 `// app.jsx`，禁止任何前言后记。
- **白名单 CDN**：仅允许 `cdn.tailwindcss.com`、`unpkg.com/react@19`、`unpkg.com/react-dom@19`、`esm.sh/lucide-react`、`unpkg.com/babel-standalone`（在服务端 prompt 与代码出站后双重校验）。
- **禁网请求**：明确禁止生成 `fetch('http(s)://...')` 调用外网；mock 数据必须内联。
- **平台风格继承**：在 prompt 中注入平台 Tailwind 主色与圆角规范，保持视觉一致。
- **可重入**：多轮修改时把 `previous_code` 注入，明确"在以下代码上进行最小修改"，避免每次重写。
- **多模态预留**：第二阶段 prompt 模板支持 `image_url` 参数（用于 F-2.11 拖入图片）。

### 5.3 与现有 `capabilities` 网关的集成

无需新增服务接口，完全复用：

- 调用入口：`POST /api/capability/one_shot_app_generator_1/stream`
- 鉴权：`@RequirePermissions('action.ai.invoke')`
- 速率限制：建议在网关补一层 `pluginId+userId` 配额（与"优化建议"文档对齐）
- 模型降级：与现有 `capabilities.service` 一致，无配置时受 `AI_DEMO_MODE` 控制

### 5.4 辅助能力插件

| 插件 | 触发场景 | 优先级 |
|---|---|---|
| `one_shot_app_fixer_1` | 预览报错时一键"让 AI 修"，专门面向修 bug 的小型 prompt | P0 |
| `one_shot_app_summarizer_1` | 生成会话标题（替代用户手动命名）& 提交需求时的摘要 | P1 |
| `one_shot_app_explainer_1` | 在代码视图悬停某段代码时给出解释 | P2 |

### 6.2 全局挂载

- 在 `web/src/app/(main)/layout.tsx` 中无侵入挂 `<AiLauncher />`；登录页 `(auth)` 不挂。
- 权限闸门：通过 `useAccessControl()` 校验 `action.ai.one_shot_app`，无权限不渲染入口。

### 6.3 状态机（Zustand）

```ts
type AppGenStatus = 'idle' | 'streaming' | 'error' | 'applying' | 'ready';

interface AppGenSession {
  id: string;                // uuid
  title?: string;            // AI 自动生成
  messages: Message[];       // 多轮对话
  currentCode: string;       // 当前版本完整代码
  versions: Array<{ id: string; code: string; createdAt: string; intent: string }>;
  status: AppGenStatus;
  contextChips: ContextChip[];
  deviceTarget: 'desktop' | 'tablet' | 'mobile';
  theme: 'light' | 'dark';
}
```

- **流式合并策略**：在 `useStreamingApp` 中按 chunk 累积；当探测到代码块完整（`</html>` 或 `// EOF`）时整体替换 `currentCode`，触发预览刷新；预览刷新通过"延迟 + 节流"避免抖动（建议 250ms idle 后刷新）。
- **回滚**：`versions` 维护历史，UI 提供"上一版本"按钮（F-4.5 Diff 同源数据）。

### 6.4 UI 复用

- Drawer：复用 `web/src/components/ui/drawer.tsx`（vaul 已有依赖）。
- Code：`shiki` + 现成 `code-block-shiki` 抽出通用 `<CodeViewer>` 组件。
- Markdown 流：使用 `streamdown`（已有依赖）渲染助手消息。
- 拖拽：`@dnd-kit/core`（已有）做悬浮按钮拖拽。
- 状态色：复用平台 6 阶段色（仅作辅助），主色 / 圆角 / 阴影遵循 `AGENTS.md`。

## 七、沙箱预览的安全方案

### 7.1 渲染机制

- 使用 `<iframe srcDoc={renderedHtml} sandbox="allow-scripts" />`，**不加** `allow-same-origin`、**不加** `allow-top-navigation`。
- 这样 iframe 内代码：
  - 可以执行 JS（满足交互需要）
  - 无法访问父窗 `localStorage` / Cookie / `window.parent`
  - 无法跳转父窗
  - 跨域，因此无法读取 `document.cookie`、`window.opener`

### 7.2 内容安全策略（CSP）

`srcDoc` 顶部注入 meta：

```html
<meta http-equiv="Content-Security-Policy" content="
  default-src 'none';
  script-src 'unsafe-inline' 'unsafe-eval' https://cdn.tailwindcss.com https://unpkg.com https://esm.sh;
  style-src  'unsafe-inline' https://cdn.tailwindcss.com;
  img-src    data: blob: https:;
  connect-src 'none';
  font-src   https: data:;
  frame-src  'none';
">
```

- `connect-src 'none'`：阻止 iframe 内代码发起任何外网请求（防数据外发 + 防钓鱼）。
- `script-src` 显式白名单 CDN 域，与 prompt 端约束保持一致。

### 7.3 代码出站校验（服务端）

服务端在 stream 完成时做一次轻校验（仅警告，不阻断 MVP）：

- 禁止：`fetch(http`、`new WebSocket`、`navigator.sendBeacon`、`document.cookie`、`window.parent`、`window.top`、`eval(`（除 babel-standalone 外）。
- 命中告警 → 客户端在代码视图标黄、运行前给出提示。

### 7.4 父子通信（postMessage 桥）

- 子 → 父：转发 `console.*`、`onerror`、`unhandledrejection`，落 F-3.4 / F-3.5。
- 父 → 子：切换主题、改设备宽、注入挂载数据。
- 通信通道 `origin` 严格校验，message 走签名 `nonce`，防误触。

### 7.5 用户提交前的二次确认

「提交到需求」「推到 Agent 工作区」前弹确认框，明确告知"以下代码将被存档并供下游使用，请确认无敏感信息"。

## 十二、安全与合规风险及对策

| 风险 | 对策 |
|---|---|
| **XSS / DOM 劫持** | sandbox iframe + 严格 CSP + 禁 `allow-same-origin`（第七章） |
| **数据外发** | `connect-src 'none'`；服务端代码出站正则告警；用户提交前确认 |
| **Prompt 注入** | 用户输入按 markdown 转义后注入 prompt；指令与数据分离；助手系统消息固化优先级 |
| **AI Key 泄露** | 严格走 `/api/capability/*`，前端不引入任何 `NEXT_PUBLIC_ARK_*`（与优化建议安全章对齐） |
| **生成代码侵权** | 提交需求 / PRD 前在 UI 提示"由 AI 生成，使用前请审阅"，并落到 `audit_logs` |
| **敏感词与 PII** | 入口侧调用前先过 PII 红线（复用优化建议中 `pii-redactor`） |
| **滥用刷量** | 网关层 rate-limit；超过日配额降级到本地小模型或弹出提示 |
| **分享链接外泄** | 默认带 6 位口令 + 24h 过期 + 仅 GET；可追溯访问 IP |

## 十五、验收标准（关键）

1. **入口可达**：在 9 大主模块任一页面均可 ≤ 1 次点击或 1 个快捷键触达。
2. **首版有效率**：首轮生成的代码可成功在沙箱渲染的比例 ≥ 90%。
3. **多轮稳定性**：连续 10 轮对话不丢上下文、不超时；显存型错误自动降级且有提示。
4. **预览安全**：第三方安全扫描确认沙箱无 `allow-same-origin`；CSP `connect-src` 阻止任意外网请求。
5. **沉淀闭环**：提交到需求的版本可在需求详情打开预览，并可追溯到原会话。
6. **可观测**：每次生成的 token 数、延迟、模型、状态都在 `ai_invocation_log` 中可查。
7. **权限**：未授权角色 100% 不可见入口；后端接口拒绝越权请求。
8. **可中断**：流式过程中点击"停止"≤ 300ms 切回 idle 状态。
